# Fact → Pattern → Remedy Mapping

## Fact (Mirror)
Single verifiable instance with sources.

## Pattern (Water)
Two or more facts that show repetition across time.

## Remedy (Fire)
A practical solution that:
- is specific
- is measurable
- is child-centered
- reduces future disputes

## Example
Fact: missed exchange on date X (EXH-010)
Pattern: missed exchanges on X, Y, Z (EXH-010, EXH-014, EXH-021)
Remedy: define exchange protocol + make-up time window + communication channel rules
